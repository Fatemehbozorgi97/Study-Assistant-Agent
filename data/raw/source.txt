MLP for Regression

MLPs can be used for regression tasks.
The number of output neurons depends on the prediction problem:

Univariate regression → 1 output neuron
Multivariate regression → multiple output neurons

Typical examples:

House price prediction
Coordinate prediction
Continuous value estimation

Output activation choices:

No activation → unrestricted outputs
ReLU / Softplus → positive-only outputs
Logistic / Tanh → bounded outputs

Loss functions:

MSE
MAE
Huber loss

An MLP (Multi-Layer Perceptron) for regression is a standard neural network setup used when the goal is to predict continuous values instead of classes.

What an MLP is doing in regression

An MLP is a stack of fully connected layers that learns a mapping:

input features→continuous output(s)

It transforms inputs through hidden layers using nonlinear activations, and then produces a numeric prediction at the end.

This is widely used in Machine Learning for tasks like prediction and function approximation.

Output layer design (very important)

The output layer depends on what you are predicting:

1. Univariate regression
Predict one value
Example: house price, temperature

👉 Output layer:

1 neuron
2. Multivariate regression
Predict multiple values at once
Example: (x, y, z) coordinates, multi-property prediction

👉 Output layer:

n neurons, one per target variable
Output activation choices

Unlike classification, regression outputs are continuous, so activation choice matters:

1. No activation (linear output)
Most common choice
Output can be any real number

✔ Best for:

house prices
general continuous values
2. ReLU / Softplus (positive-only outputs)
Ensures outputs are ≥ 0

✔ Useful when:

predicting counts
prices (sometimes)
physical quantities that cannot be negative

Softplus is smoother:

softplus(x)=log(1+e
x
)
3. Sigmoid / Tanh (bounded outputs)
Sigmoid → (0, 1)
Tanh → (-1, 1)

✔ Useful when:

output is normalized
predicting probabilities or scaled values
Loss functions for regression

The model learns by minimizing error between prediction and true value:

1. MSE (Mean Squared Error)
Penalizes large errors more strongly
Most commonly used
2. MAE (Mean Absolute Error)
More robust to outliers
Linear penalty on error
3. Huber loss
Combination of MSE and MAE
Smooth for small errors, robust for large ones
Intuition summary

An MLP for regression:

Takes inputs (features)
Learns nonlinear relationships via hidden layers
Outputs continuous value(s)
Is trained by minimizing a regression loss

🔹 Hidden neurons (width)

🧠 2. Training Neural Networks

Now we move from biology inspiration → actual machine learning process.

🔁 Training loop (core cycle)

Training a neural network always follows this loop:

1. Draw training batch
Take a small subset of data (mini-batch)
Not the whole dataset at once
2. Forward pass
Input goes through the network
Produces predictions
3. Compute loss
Compare prediction vs true value
Measure error
4. Update weights
Adjust parameters to reduce error
Done using optimization (usually gradient descent)
⚙️ What is being learned?

The network learns two types of parameters:

🔹 Weights (W)
control strength of connections
decide how much influence each input has
🔹 Biases (b)
shift outputs
allow flexibility even when inputs are zero
🎲 Initialization

At the start:

weights are randomly initialized

Why?

If all weights were equal → network would learn nothing useful
randomness breaks symmetry
📉 Learning process intuition

Over time:

random weights → bad predictions
compute loss → measure error
adjust weights → reduce error
repeat thousands of times

Eventually:

the network “shapes itself” into a function that maps inputs → outputs well

Gradient Descent (GD)

Gradient Descent updates weights iteratively to minimize loss.

Key idea:

compute gradient
move parameters opposite the gradient direction

This reduces error over time.

Variants:

SGD
SGD with momentum
RMSProp
Adagrad

Momentum improves:

convergence speed
local minima handling

📉 Gradient Descent (GD)

Gradient Descent is an optimization method used to minimize the loss function of a neural network.

In simple terms:

It tells the model how to change its parameters so predictions become better.

🧠 Core idea

You have a loss function:

high loss → bad model
low loss → good model

Gradient Descent answers:

“In which direction should we change weights to reduce the loss fastest?”

📌 Key idea (your note)
✔ Compute gradient

The gradient tells you:

slope of the loss function
direction of steepest increase
✔ Move opposite the gradient

So we do:

move parameters in the direction that reduces loss

Mathematically:

W=W−η⋅∇L(W)

Where:

W = weights
η = learning rate
∇L(W) = gradient of loss
🎯 Intuition

Imagine you're on a mountain (loss landscape):

height = loss
goal = reach lowest point

Gradient Descent:

looks around
finds steepest uphill direction
steps downhill instead
🔁 Why it is iterative

You don’t reach the minimum in one step.

Instead:

compute gradient
take a small step down
recompute
repeat many times

This gradually improves the model.

⚡ Variants of Gradient Descent

Different versions exist to make learning faster and more stable.

1. 🟦 SGD (Stochastic Gradient Descent)

Instead of using the full dataset:

uses one sample or small batch

✔ Pros:

fast updates
works well for large datasets

❌ Cons:

noisy updates
can bounce around
2. 🟩 SGD + Momentum

This improves SGD by adding “memory”.

Idea:

Instead of only using current gradient:

also remember past updates

Think of it like:

a ball rolling downhill that gains speed

Why it helps:

✔ faster convergence
✔ smoother updates
✔ reduces zig-zag motion
✔ helps escape shallow local minima

3. 🟨 Adagrad

Adapts learning rate per parameter:

frequent updates → smaller steps
rare updates → larger steps

✔ Good for:

sparse data (like NLP)

❌ Problem:

learning rate keeps shrinking too much over time
4. 🟧 RMSProp

Fixes Adagrad issue:

uses moving average of squared gradients
keeps learning rate stable

✔ Very popular for:

deep learning
RNNs
🧠 Big picture intuition

All GD variants try to answer:

“How do we move weights to reduce loss efficiently and stably?”

Differences are mainly about:

how much data we use
how we scale learning rate
whether we use memory of past gradients
📊 Simple comparison
Method	Key idea	Strength
GD	full dataset	stable but slow
SGD	one batch	fast but noisy
Momentum	adds memory	smoother + faster
Adagrad	adaptive learning rate	good for sparse data
RMSProp	fixes Adagrad decay	stable deep learning
🧠 Final intuition (important)

Training neural networks is basically:

start with random weights
compute error (loss)
compute slope (gradient)
move weights downhill
repeat

👉 Gradient Descent is the rule that decides how to move

🚶 1. Stride (how fast the filter moves)

Stride controls how much the convolution window moves each step.

📌 Intuition

Think of sliding a magnifying glass over an image:

stride = 1 → move 1 pixel at a time (slow, detailed scan)
stride = 2 → jump 2 pixels (faster, less detail)
📊 Effects of stride
🔹 Larger stride:
smaller output feature maps
fewer computations
less overlap between receptive fields
🔹 Smaller stride (default = 1):
more detailed feature maps
more overlap
higher computational cost
🧠 Key idea

stride controls the resolution of the output feature map

🧱 2. Padding (controlling image size)

Padding adds extra pixels around the input (usually zeros).

📌 Why padding is needed

Without padding:

image shrinks after every convolution
edge information gets lost quickly
🧠 Intuition

Padding is like:

adding a border so the filter can “see” edges properly

📊 Effects of padding
❌ No padding (valid convolution)
output gets smaller every layer
edges are underused
✅ With padding
preserves spatial size
keeps edge information
🧩 Types of padding
🔹 1. Valid padding
no padding added
output shrinks

👉 most “strict” version

🔹 2. Same padding
output size ≈ input size
pads just enough to preserve dimensions

👉 most commonly used in deep CNNs

🔹 3. Full padding
adds extra border so output becomes larger
less common in modern CNNs
🧠 Key idea

padding controls whether spatial dimensions shrink or stay stable

📉 3. Pooling layers

Pooling reduces spatial size while keeping important information.

📌 Why pooling is used

It helps:

reduce computation
prevent overfitting
make model robust to small shifts
🧠 Intuition

Pooling is like:

compressing an image while keeping the most important signals

🔷 Most common: Max Pooling
📌 How it works

Take a window (e.g. 2×2) and keep only the maximum value.

Example:

🔹 2. Reduced overfitting

🌳 1. Ensemble Learning

Ensemble learning means:

instead of using one model, we combine many models to get a better one

🧠 Core idea

A single model can make mistakes.

But:

multiple models can “vote” and correct each other

So performance improves.

📦 Main methods
🗳️ 1. Voting

Used in classification.

🔹 Hard voting

Each model votes for a class:

Example:

model 1 → A
model 2 → B
model 3 → A

👉 final answer = A (majority wins)

🔹 Soft voting

Instead of labels, use probabilities:

Example:

Model	A	B
M1	0.7	0.3
M2	0.6	0.4
M3	0.9	0.1

Average:

A = 0.73
B = 0.26

👉 final prediction = A

🧠 Why soft voting is better

Because:

it uses confidence
strong models influence more
smoother decision making
🌲 2. Bagging

👉 Train models independently on random subsets of data

reduces variance
stabilizes predictions

Example:

Random Forest
🚀 3. Boosting

👉 Models are trained sequentially

Each model:

fixes errors of previous ones

Examples:

AdaBoost
XGBoost
🧩 4. Stacking

👉 Combine multiple models using a “meta-model”

Flow:

base models make predictions
meta-model learns how to combine them
🧠 Wisdom of the Crowd

This is the theoretical foundation:

many weak learners together can form a strong learner

BUT only if:

✔ diversity

models must be different

✔ uncorrelated errors

they should not fail in the same way

💡 Key intuition

If everyone makes different mistakes:

majority vote cancels errors
